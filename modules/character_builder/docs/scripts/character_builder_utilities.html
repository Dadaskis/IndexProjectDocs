CharacterBuilderUtilities singleton class contains shared logic
in form of utility functions. It is mainly used to arrange texture data
as well as load images and create materials

Constant

SAVED_TEX_NAMES

[]

A list of "saved" texture names, used by save_texture_data.
It is required to properly save subtextures in case if they were changed

Function

validate_materials

mesh: Mesh, path: any

any

Validates and updates materials for a given Mesh
Ensures all materials are properly configured and loaded from files when available,
with handling for UserSpatialUbershader materials to apply albedo_correction effection.

Function

load_image_texture

any

Loads an image texture with provided path

Function

check_mesh_materials

mesh: MeshInstance, res_path: String

any

Checks and updates materials for a MeshInstance by:
1. Iterating through each surface material in the mesh
2. Creating a new UserSpatialUbershader material with default PBR settings
3. Auto-assigning textures based on filename suffixes (_D, _N, _MRSAO, etc.)
4. Enabling relevant material features (albedo, normal maps, emission, etc.)
5. Applying neutral albedo correction effect
6. Assigning the new material back to the mesh surface

Eventually this function is used to replace manual creation of the materials
with runtime loading of textures

Function

get_subtextures_dict

path: String, mesh: Mesh

any

Scans a directory for texture files and organizes them into a dictionary structure:
1. Parses subdirectories in format " [Material] __ [Subtexture] " to identify material variations
2. Loads textures based on suffix (_D=albedo, _N=normal, _MRSAO=PBR map)
3. Also captures existing textures from the mesh's materials as "default" variants
Returns a nested dictionary structure: {mat_name: {subtex_name: {texture_type: texture} } }
Example output:
{"Concrete": {"wet": {"albedo_tex": tex1} , "default": {"normal_map_tex": tex2} } }

Function

save_texture_data

mesh: Mesh

any

Saves all texture paths and material properties
from a Mesh's materials into a dictionary.
For each material surface:
1. Stores paths of all textures (albedo, normal, etc.) referenced by the material
2. Captures all float, color, and flag properties
3. Records enabled/disabled states for material effects
Returns a dictionary structured as: {material_name: {property_name: value} }
Example output:
{"MetalRusty": {"albedo_tex": "res://tex/metal_d.png", "roughness": 0.8} }

Function

load_texture_data

mesh: Mesh, dict: Dictionary

any

Loads texture paths and material properties
from a dictionary back into a Mesh's materials.
For each material surface:
1. Applies saved texture paths by loading the textures and assigning them
2. Restores all saved float, color, and flag properties
3. Triggers a material update to apply changes
Expects dictionary format: {material_name: {property_name: value} }
Example input:
{"MetalRusty": {"albedo_tex": "res://tex/metal_d.png", "roughness": 0.8} }
