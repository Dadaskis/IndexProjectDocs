Description

So, let's assume that you've started making a game and you have for about zero clues how you would organize your game levels in a game engine that gives you nothing but scenes and says "Now this is your bread and you will eat it everyday". First thought that I had in my mind - hey, I can write a SceneChanger singleton to load *any* scene. With a fancy ass loading screen, too! It would be amazing. It would be something that would make one happy. So I did. Then... Time passed... I made LocationChanger ... SceneChanger and LocationChanger have a co-dependent relationship that harms both... And here we are, having a weird kind of technical debt that is playing 4D-chess with my mind. Practically, SceneChanger should work just fine, you'd be able to provide any scene path and it'll handle it just well. But really? I'm not sure. I don't want to know. I don't want to check. It exists *out there*, it does *that*, it fucks with LocationChanger - alright, so be it.

Usage Example

SceneChanger.change_scene("res://path/to/your/scene.scn")
# Think twice, maybe you just need LocationChanger.change_location("game_lvl")?

SceneChanger.set_status0_text("Loading stuff...")
SceneChanger.set_status1_text("Doing that carefully...")
SceneChanger.set_status2_text("0 / 1337")

Honestly, you don't need to use that class. Except for set_statusX_text , you seriously have no reason. Don't. Please. Save your time.
