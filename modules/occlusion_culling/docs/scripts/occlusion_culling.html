An ANCIENT occlusion culling with blackjack and hookers that i wrote someday
before. It was used in build 2 as far as i reckon, however,
due to complexity of setup, long baking times, and general
problems with this system - it was left unused.
The only reason it exists in this codebase is because
it reminds me of good old days... that weren't so old and weren't so good.

How it works:

1. Divide location into chunks (CullChunk nodes with AABBs)
2. Assign meshes to chunks based on AABB intersection
3. For each chunk:
- Place camera at view points inside chunk
- - For each camera angle (6 cardinal directions):
- - - Render scene with target chunk visible
- - - Check rendered image for non-black pixels
- - - If visible, mark chunk as potentially viewable
4. Save visibility relationships between chunks
5. At runtime:
- When camera enters chunk:
- - Show meshes in current chunk
- - Show meshes in chunks marked as visible
- - Hide all other meshes

Uses viewport rendering with black materials to test visibility
between chunks from multiple angles and positions, hence why it is
slow as shit.
