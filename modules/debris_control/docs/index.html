Description

Contains only DebrisControl singleton that is being a global queue with extra steps. It was made for optimization of debris, it removes excessive amount of those. If overall amount of debris goes beyond the limit, the oldest pieces will be removed first.

Usage Example

if not DebrisControl.can_create_debris(): # Stops initialization if there's too much debris
 return
DebrisControl.add_debris(self)     # Adds debris

All debris nodes need to have a remove_debris method that will be called upon their "removal". DebrisControl itself is NOT removing any nodes because they might need special treatment (aka removing from a scene tree instead of queue_free ing them).
