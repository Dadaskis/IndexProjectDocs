Description

This is an amazing module to work with sound. I mean, it doesn't do anything special in particular and mostly it is all about bullshit that is happening inside Sound , but you know what...

What does Sound do?
It's basically a singleton to manage sound in this game. You know, volumes, some SFX. Here's the list:

Sound Type Management:
 - Categorizes sounds into types (environmental, music, voice lines, etc.)
 - Handles different playback behaviors for each type
 - Automatic bus routing based on sound type

Spatial Audio:
 - Supports both 2D and 3D sound playback
 - Obstacle detection for sound occlusion (commented out because it was so damn cursed)

Environmental Effects:
 - Dynamic reverb system that analyzes the environment (mainly unused?)
  - Manual configuration in soundscapes might be more preferrable
 - Automatic environment classification (hallway, small room, outdoor, etc.)
 - Smooth transitions between different acoustic environments
 - Special underwater audio processing with filters

Sound Playback:
 - Creation and playback of sounds with various parameters
 - Pitch modulation for varied sound effects
 - Automatic cleanup of finished sounds
 - Smooth fade-in/out effects
 - All in code

Volume Management:
 - Individual volume controls for different sound categories
 - Master volume control
 - Voice ducking (music volume reduction when voice lines play)
 - Special effect volumes (flashbang, underwater)

Event Integration:
 - Checkpoint system integration for sound state persistence

Special Effects:
 - Flashbang audio effects with temporary suppression
 - Underwater audio filtering

remove_sound_on_finish.gd - Simple helper node script to remove a sound when its done playing
location_sound_2D.gd and location_sound.gd - Nodes that extend functionality of AudioStreamPlayer and AudioStreamPlayer3D to include additional functionality required for whatever reason.
reverb_system.gd - A cursed automatic detection system for reverb, it's really weird and never worked that well in my opinion.
sound.gd - The singleton that made this entire module real, its main purpose is related to control of SFX in code, which includes bus management and creation of new sounds fast.
