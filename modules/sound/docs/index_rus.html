Description

Это потрясающий модуль для работы со звуком. Ну, то есть он не делает ничего особенного конкретного, и в основном всё сводится к той хрени, что творится внутри Sound , но знаешь что...

Что делает Sound?
Это по сути синглтон для управления звуком в этой игре. Ну знаешь, громкости, какие-то SFX. Вот список:

Управление типами звуков:
 - Классифицирует звуки по типам (окружение, музыка, голосовые реплики и т.д.)
 - Обрабатывает различное поведение воспроизведения для каждого типа
 - Автоматическая маршрутизация по шинам на основе типа звука

Пространственный звук:
 - Поддерживает воспроизведение как 2D, так и 3D звуков
 - Обнаружение препятствий для окклюзии звука (закомментировано, потому что было чертовски проклято)

Эффекты окружения:
 - Динамическая система реверберации, которая анализирует окружение (в основном не используется?)
  - Ручная конфигурация в soundscapes может быть более предпочтительной
 - Автоматическая классификация окружения (коридор, маленькая комната, улица и т.д.)
 - Плавные переходы между различными акустическими средами
 - Специальная обработка подводного звука с фильтрами

Воспроизведение звуков:
 - Создание и воспроизведение звуков с различными параметрами
 - Модуляция высоты тона для разнообразных звуковых эффектов
 - Автоматическая очистка завершённых звуков
 - Плавные эффекты fade-in/out
 - Всё в коде

Управление громкостью:
 - Индивидуальный контроль громкости для различных категорий звуков
 - Общий контроль master-громкости
 - Уменьшение громкости музыки при воспроизведении голосовых реплик
 - Громкости специальных эффектов (вспышка, под водой)

Интеграция с событиями:
 - Интеграция с системой чекпоинтов для сохранения состояния звука

Специальные эффекты:
 - Аудио эффекты вспышки с временным подавлением
 - Фильтрация подводного звука

remove_sound_on_finish.gd - Простой скрипт ноды-помощника для удаления звука, когда он закончил воспроизведение
location_sound_2D.gd и location_sound.gd - Ноды, которые расширяют функциональность AudioStreamPlayer и AudioStreamPlayer3D , чтобы включить дополнительную функциональность, нужной по какой-то причине.
reverb_system.gd - Проклятая система автоматического обнаружения реверберации, она очень странная и никогда не работала так нормально, по моему мнению.
sound.gd - Синглтон, который сделал весь этот модуль реальным, его обязанность связана с контролем SFX в коде, что включает управление шинами и быстрое создание новых звуков.
