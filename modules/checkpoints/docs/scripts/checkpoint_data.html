Resource class for storing and managing game checkpoint/save data

Encapsulates all data needed for game saves/checkpoints.
It handles both the metadata (location, difficulty, etc.) and the actual game state data.

Key Features:
- Stores both game state (in the data Dictionary) and visual/session information
- Manages screenshot images with fallback to default image when needed
- Provides easy texture access through get_image_texture()
- Serializable as a Resource for easy saving/loading

Note: This is typically used in conjunction with the Checkpoints autoload
which manages the actual save/load operations and maintains the save list.
