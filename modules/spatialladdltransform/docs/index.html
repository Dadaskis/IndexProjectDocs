Description

So, sit down, feel yourself comfortable, and listen to my story... Let's assume that you are making a hyper-complex player controller that is going to kick anyone's ass. Hell yeah. Feeling good. Anyway, there's a problem... A lot of things that make your controller "alive" are related to camera movements. So, how would you handle this all without turning it into a messy spaghetti that'll shoot your legs off? **DING-DING-DING** you are right! This is what this module is all about! It wasn't really a nice story, but you get the point.

SpatialAddTransformManager is a node that is added as a child of any Spatial -extended node. It will apply some magic tricks of offsets to the parent. All of the changes are additive and are controlled through SpatialAddTransformData class. Simply saying, you use SpatialAddTransformManager to register SpatialAddTransformData and then you change its data to control offsets. This way you can have a lot of factors that can affect your potential camera's view without doing it in a destructive way. Of course, it was made for cameras in mind, but it can be used for any Spatial -extending node because, like, we are software engineers, loving extensible things that does stuff outside its original scope, right?.. Right?..

Usage example

var camera: Camera # Let's assume you have one
# ...

var manager = SpatialAddTransformManager.new()
camera.add_child(manager)
manager.initialize()

# ...

var data: = manager.register_data(true)
# ^^^ "true" argument is needed to immediately add it to the list
data.degrees_rotation = Vector3(30.0, 0.0, 0.0)

# Tada, it should affect your transforms

Check documentation of SpatialAddTransformData and SpatialAddTransformManager to find out what else they can do.

For more information, please reread.
