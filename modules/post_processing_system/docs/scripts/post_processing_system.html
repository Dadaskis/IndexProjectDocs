Script: post_processing_system

Class Name: PostProcessingSystem

Extends: Object

CLASS

Page last updated:

Manages visual post-processing effects in the game.
Handles effect registration, updates, and lifecycle management.
Each effect runs on its own CanvasLayer for proper rendering order.

Constant

EFFECTS_PATH

"res://modules/post_processing_system/assets/effects/"

Path to the directory containing post-processing effect resources

Constant

EFFECT_FILE

"effect.tres"

Default filename for effect resource files

Variable

owner

Undefined/Default

Reference to the owner node that will contain the effect layers

Variable

effects

[]

Array of all active post-processing effects

Variable

effects_dict

{}

Dictionary of effects keyed by their names for quick lookup

Variable

id_counter

0

Counter for generating unique IDs for effects

Function

owner: Node

any

Initializes the system with the node that will own the effect layers
owner: The parent node that will contain the CanvasLayers

Function

delta: float

any

Updates all registered effects every frame

Function

delta: float

any

Updates process-dependent logic for all effects
delta: Time elapsed since last frame (in seconds)

Function

any

Function
