AUTOLOAD

Page last updated:
November 2, 2025 at 8:59 UTC


Singleton that manages initialization of game systems during startup.
In standalone builds, queues initialization calls to display during loading screen.
In editor/debug mode, immediately executes initialization calls.

Variable

data

[]

Array of initialization data containing:
[0] - Funcref to initialization method
[1] - Display name for loading screen

Function

add_init

obj: any, method_name: any, loading_name: any

any

Adds an initialization method to the startup sequence
obj: The object containing the initialization method
method_name: Name of the initialization method to call
loading_name: Display text for loading screen
