Description

This one was made entirely by TNP so... Not gonna be too edgy in this one... Look, you probably played a lot of games that have a "texture quality" setting, right? A pretty pointless one if you ask me. Either way, it does exist and helps to fight videomemory usage. Yeah. Wouldn't that be great if Godot already had it? Yeah. But it doesn't. And so what? WE ARE GOING BALLS DEEP BABY WE ARE PROCESSING THIS SHIT OURSELVES HELL FUCKING YEAH.

Despite the name of the module, "texture changing" operation here is just about resizing all textures to a different size. That's all you need to know.

Usage example

TextureChangerServer.add_location_mesh(mesh)
TextureChangerServer.add_character_mesh(mesh)
TextureChangerServer.add_viewmodel_mesh(mesh)
# By adding a mesh you register all the needed textures fast

# Update all textures like a real dude
TextureChangerServer.update_all_textures()
