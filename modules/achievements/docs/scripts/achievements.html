Main singleton to work with achievements.

Note: Achievement states are persisted through the Values singleton
- "achievements_unlocked" stores boolean unlock states
- "achievements_counts" stores incremental progress counts
That simply means they aren't tied to any of the save files and are fully
global in terms of this game.

Achievement resources should be placed in:
res://modules/achievements/assets/data/
and must be of type AchievementData (.tres resources)

Usage example

Achievements.unlock("ach_name") # Unlock an achievement called "ach_name"
Achievements.lock("ach_name") # Lock that achievement back
Achievements.add_count("ach_name") # Adds one "count" to an achievement
Achievements.add_count("ach_name", 12) # Adds 12 "count" to an achievement

Keep in mind some achievements may have a "count" property that may be useful
for achievements that have iterative progress.

Signal

unlocked

ach_name

Emitted if any achievement is unlocked.
ach_name - Unlocked achievement's name.

Signal

count_changed

ach_name, amount

Emitted if achievement's count has been changed.
ach_name - Achievement's name.
amount - Achievement's current count.

void

Unlock an [ach_name] achievement.
Emits an "unlocked" signal.
Will ignore if achievement is already unlocked

Function

ach_name: any

void

Locks an [ach_name] achievement.
Also resets the achievement's progress count

Function

add_count

ach_name: any, count = 1

void

Adds [count] to count of [ach_name] achievement.
If achievements count reaches the required count from its data resource,
it will automatically unlock the achievement.
Emits "count_changed" signal.
Clamps count to maximum required by achievement data
