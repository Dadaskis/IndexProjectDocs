Module: bullet_system

Script: trail

Class Name: null

Extends: MeshInstance

NODE

Page last updated:
October 23, 2025 at 21:10 UTC


A little script for bullet trail to move. Use "initialize" to launch the
trail. Keep in mind: Both start_pos and end_pos are calculated all by
yourself.

Constant

SPEED

100.0

Speed value of the bullet trail.

Variable

start_pos

Undefined/Default

Start position that was set with "initialize"

Variable

end_pos

Undefined/Default

End position that was set with "initialize"

Variable

path_complete

Undefined/Default

How much of the path is complete? [0 .. 1] range.

Variable

path_step

Undefined/Default

Path step value that is applied on trail each frame.
Basically that's real velocity of bullet trail.

Variable

light_disabled

false

Is light disabled? This property can be used to disable lights for bullet trails
even if game menu's setting is enabled.

Variable

shader_compile

false

Is it shader compiling right now?
If shader compilation is active,
it is forcing the bullet to stay on the same location to be seen by camera.

Function

initialize

start_pos: Vector3, end_pos: Vector3, shader_compile = false

void

Initialization of bullet trail that eventually launches it.
shader_compile makes it stay still as it is said by description above.

Function

calculate_transition

path_complete: float

void

Calculates travelling of the bullet trail from A to B.
Simply saying, it is applying path_complete value
to lerp between start_pos and end_pos

Function

_physics_process

delta: any

any

Update of the bullet trail.
Doesn't run logic if it is not visible, or it is not inside of tree,
or if shader compilation is active.
Otherwise, calculates path_complete, and if path_complete > 1.0,
makes the bullet invisible and disables _physics_process
